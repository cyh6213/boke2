# RAG项目改进手册

## 项目概述

RAG（Retrieval-Augmented Generation）问答系统是一个基于大语言模型的智能问答系统，能够将用户问题与知识库中的文档进行匹配，生成精准的回答。

## 三次改进历程

### 第一次改进
- 基础功能完善
- 实现文档上传和基础检索功能
- 建立项目基础架构

### 第二次改进
- 引入混合检索策略
- 提升回答准确性
- 优化检索算法

### 第三次改进
- 增加记忆功能
- 性能优化
- 用户体验提升

## 核心模块详解

### 文件上传与解析
支持多种格式文档的上传和解析：
- PDF 文档解析
- Word 文档解析
- Markdown 文件支持
- 文本文件处理

### 文本处理与混合检索
- 文档向量化处理
- 语义检索算法
- 混合检索策略（关键词+语义）
- 上下文感知检索

### LLM交互
- 与大语言模型的交互接口
- 支持多种主流大模型
- 提示词工程优化
- 响应格式标准化

### 权限/登陆控制
- 用户认证系统
- 角色权限管理
- 文档访问权限控制
- Session 管理

### 记忆功能
- 对话历史记忆
- 上下文理解
- 多轮对话支持

## 技术栈

- **后端框架**: Spring Boot
- **数据库**: MySQL + Redis（多级缓存策略）
- **向量数据库**: Milvus/Chroma
- **实时通信**: WebSocket 长连接
- **日志系统**: 完善的日志记录与注解
- **性能监控**: 监控系统 + 压力测试

## 未来规划

持续优化检索精度和系统性能，探索更多 AI 能力集成。
